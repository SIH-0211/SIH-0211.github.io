# 📈 실시간 암호화폐 가격 추적기 (블록체인 실습)

본 레포지토리는 블록체인 실습 주간 미션을 수행하기 위해 생성된 프로젝트입니다. CoinGecko API를 활용하여 Bitcoin(BTC)과 Ethereum(ETH)의 실시간 가격 데이터를 받아오고, 이를 웹 화면에 출력하는 프론트엔드 웹페이지를 구현했습니다.

## 🚀 미션 수행 내역

### 1. GitHub 호스팅 (GitHub Pages)
* `[유저네임].github.io` 형식의 레포지토리를 생성하여 정적 웹페이지 무료 호스팅을 구축했습니다.
* 로컬 환경에서 테스트한 HTML 파일을 원격 저장소에 Push하여 배포를 완료했습니다.

### 2. 정적 가격 테이블 작성 (Markdown to HTML)
* CoinGecko 웹사이트에서 수동으로 BTC와 ETH 가격을 확인 후 `price_record.md` 마크다운 파일로 기록했습니다.
* 생성형 AI(LLM)를 활용하여 작성된 마크다운 표를 HTML 코드로 자동 변환하는 실습을 진행했습니다.

### 3. 실시간 가격 연동 (CoinGecko API & JavaScript)
* JavaScript의 `fetch` 함수를 사용하여 CoinGecko REST API(`/simple/price`)를 호출했습니다.
* `setInterval`을 이용해 페이지 새로고침 없이 **60초 주기**로 백그라운드에서 데이터를 가져와 화면의 테이블을 실시간으로 갱신하도록 구현했습니다.

### 4. UI/UX 개선
* **다크 테마(Dark Theme) 적용:** 실제 암호화폐 거래소와 유사한 어두운 배경(`#121212`)과 밝은 텍스트를 적용하여 가독성을 높였습니다.
* **중앙 정렬:** CSS의 `margin: auto` 속성을 활용하여 테이블을 화면 중앙에 깔끔하게 배치했습니다.

## 🛠 트러블슈팅 및 문제 해결 과정 (Troubleshooting)

개발 및 테스트 과정에서 발생한 이슈들과 이를 해결하기 위해 적용한 엔지니어링 솔루션입니다.

### Issue 1: API 호출 제한 (HTTP 429 Too Many Requests)
* **상황:** 로컬 환경에서 과도한 새로고침 및 API 호출로 인해 CoinGecko 서버로부터 IP 기반 Rate Limit 차단 발생.
* **해결:** 호출 주기를 60초로 늘려 서버 부하를 줄이고 정책을 준수하도록 코드를 수정했습니다.

### Issue 2: 데이터 갱신 실패 (브라우저 캐싱 문제)
* **상황:** 시간(마지막 업데이트)은 갱신되지만, 실제 가격 데이터가 최초 1회 호출된 이후 변하지 않는 현상 발생. 
* **해결 (Cache Busting):** 브라우저가 동일한 URL 요청에 대해 과거의 데이터를 재사용(캐싱)하는 것이 원인임을 파악했습니다. 요청 URL 끝에 `new Date().getTime()`을 활용한 타임스탬프 파라미터(`_t`)를 붙이고, `fetch` 옵션에 `{ cache: 'no-store' }`를 추가하여 매 호출마다 강제로 최신 데이터를 받아오도록 수정했습니다.

### Issue 3: CORS 및 로컬 환경 차단 (HTTP 403 / 데이터 갱신 실패)
* **상황:** 주소창을 통한 직접 접속은 가능하나, 코드 내 `fetch` 실행 시 브라우저의 CORS 정책 또는 API 제공자의 봇(Bot) 방지 시스템에 의해 호출이 막히는 현상 발생.
* **해결:** 로컬 테스트 시 발생하는 로컬 주소(`127.0.0.1`, `file://`) 차단 문제를 인지하고, 최종 결과물을 GitHub Pages 실제 웹 환경에 배포하여 정상적인 Origin으로 우회함으로써 통신 에러를 해결했습니다.
